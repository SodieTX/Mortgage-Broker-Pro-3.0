Views & Reports

Key Views

  • EffectiveCoverage - Resolves program vs lender coverage precedence
  • vw_program_printable - Flattened view for Excel/PDF exports
  • MissingCriteria - Shows data gaps for quality improvement
  • PendingFeedChanges - Automated changes awaiting review

Supporting Tables

  • Saved_Views - User-saved filter sets (e.g., "TX Bridge Lenders")
  • DocTemplate - PDF/Excel templates for branded exports
  • DataSensitivity - PII flagging for privacy compliance
  • Presentation - Borrower term sheet generation tracking
  • Scenario_Snapshot - Version freeze for historical accuracy

Key Design Principles

  1. No Hard Deletes - Triggers prevent deletion; use Active flags
  2. Everything Versioned - Programs, Cards, Rules all have versions with Valid_From/To
  3. AI-Native - Every table tracks Created_By (User or AI) with full audit logs
  4. Soft vs Hard Boundaries - Criteria have both marketing and absolute limits
  5. Metro Granularity - Sub-state geographic restrictions fully supported
  6. Future-Proof - Tokenization, streaming facts, multi-currency ready
  7. Incomplete Scenarios OK - Weighted scoring, not binary matching
  8. Dynamic Questions - Conditional logic engine adds/removes questions contextually

Critical Implementation Notes

  • Programs use composite keys (Program_ID, Program_Version) - all FKs must respect this
  • Scenario_Snapshot freezes exact versions used - critical for compliance
  • EffectiveCoverage view must be used for all geographic matching - never query tables directly
  • All monetary fields include Currency_Code - even if USD-only initially
  • JSONB fields have GIN indexes for performance
  • Row_Version on Scenarios prevents concurrent edit conflicts
  • Hard delete triggers are mandatory in production
